# Claude Code hook harness profile

Symphony has a **team-committed** Claude Code hook harness at `.claude/settings.json`
with four scripts under `.claude/hooks/`, generated by the `personalize-harness` skill
[source: wiki/raw/sessions/2026-06-12-claude-code-harness.md#durable-facts]. It is the
Claude Code sibling of the Pi harness in [analyses/personal-harness-pi-profile.md](personal-harness-pi-profile.md);
both encode the same `CLAUDE.md` live-infra safety posture against different agent
runtimes. Hooks fire only inside Claude Code sessions, never in `symphony-host.service`.

## Hooks and posture

- **`validate-syntax.sh`** — `PostToolUse` `Edit|Write|MultiEdit`, blocking (exit 2).
  Parse-checks the edited file: Python `py_compile`, JSON `jq .`, shell `bash -n`
  [source: .claude/hooks/validate-syntax.sh].
- **`block-bash-pattern.sh`** — `PreToolUse` `Bash`, blocking. Blocks recursive deletes
  of `/`, `~`, `$HOME` (boundary-anchored — safe absolute deletes like `rm -f /tmp/x`
  are allowed) and wrong package managers (`pip`/`pipenv`/`poetry` install/add), while
  explicitly allowing `uv pip install` and `uv add`
  [source: .claude/hooks/block-bash-pattern.sh].
- **`pre-git-checks.sh`** — `PreToolUse` `Bash`, gated to `git commit`/`git push`,
  blocking. Runs `ruff format --check` + `ruff check` on **staged `*.py` only**, then
  `uv run pytest -q` over the full suite; each check time-boxed at 180s
  [source: .claude/hooks/pre-git-checks.sh].
- **`reinject-rules.sh`** — `SessionStart` `compact`, advisory. Re-injects the
  live-infra invariants after a context compaction
  [source: .claude/hooks/reinject-rules.sh].

## Key decisions

- **Ruff runs at commit-time on changed files only**, not per-edit. The repo has no
  `[tool.ruff]` config, so repo-wide `ruff format --check .` is red (38/82 files would
  reformat, 5 lint errors); gating staged files only sidesteps the legacy backlog and
  needs no config [source: wiki/raw/sessions/2026-06-12-claude-code-harness.md#decisions].
- **The runnable test command is `uv run pytest`, not bare `python3 -m pytest`.** System
  `/usr/bin/python3` lacks `alembic` (and other deps); the project venv `.venv` has
  them, so `uv run pytest` is green (615 passed/1 skip/53s) while bare `python3 -m pytest`
  fails at collection. `CLAUDE.md`'s "Quick Checks" line was corrected to `uv run pytest`
  this session — see C-0131
  [source: wiki/raw/sessions/2026-06-12-claude-code-harness.md#decisions].
- **alembic is project-only and must not be installed system-wide.** It is already a
  declared dep (`pyproject.toml:7`), locked, and installed in `.venv` at 1.18.4
  [source: wiki/raw/sessions/2026-06-12-claude-code-harness.md#decisions].
- Path guard (cat 8) and Stop self-review were skipped by operator choice; secrets live
  outside the repo at `symphony-host.env` and in-repo `.env` is gitignored
  [source: wiki/raw/sessions/2026-06-12-claude-code-harness.md#decisions].
