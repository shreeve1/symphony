02

Hard edges

Constraint Mapping

CM-2

Hard constraint — isComposing / keyCode 229 must not submit. This blocks CJK, Vietnamese, and other IME users from accidental submit on Enter.

CM-5

Hard constraint — slash picker combobox item selected by Enter must not also trigger submit. This means picker Enter must stopPropagation before shortcut reads.

CM-11

Focus constraint — shortcut must only fire when focus is *inside* the composer textarea. Outer-page Cmd+Enter must not silently open submit.

CM-20

Soft constraint — discoverability. User must learn shortcut exists. Inline hint near submit button (e.g. '⏎ Create' label) is the cheapest discoverability lever.

03

Whose keys?

Role Playing

RP-4

Screen-reader user (NVDA/JAWS): aria-keyshortcuts='Control+Enter' on textarea announces the binding. Without it, shortcut is invisible to AT users.

RP-5

Voice-control user (Dragon): cannot speak 'press Command Enter' reliably. They need a labeled button or numeric mnemonics — shortcut is no help here.

RP-13

Mobile user (iPad with Magic Keyboard): Cmd+Enter works in Magic Keyboard but not on touchscreen. Button must remain primary on touch.

RP-20

Sysadmin customizing app for org: needs no shortcut customization at MVP. Org-level config is YAGNI; per-user config is even more so.

04

Axes & choices

Morphological Analysis

MA-3

Scope axis — global page vs component-scoped (modal/flyout). Component-scoped wins — global is a power-user feature, not MVP.

MA-6

Feedback axis — silent submit vs brief button highlight vs toast vs animation. Brief button highlight (200ms pulse) is the cheapest discoverable feedback.

MA-13

Test axis — manual QA only vs jsdom unit vs Playwright integration vs visual regression. jsdom unit for handler; Playwright for end-to-end with composition simulation.

MA-20

Architecture axis — duplicate handler per composer vs lift to SlashPickerTextarea vs custom hook. Custom hook 'useComposerShortcut' called by both composers is the cleanest seam.

Break it first

Reverse Brainstorming + Cursed Genie

RB-2

Failure: shortcut submits blank description because guard list was duplicated and one path skipped 'description.trim() !== '''. Fix — single source of truth for guards.

RB-8

Failure: macOS Cmd+W closes tab while user expected shortcut to send; app loses draft. Fix — do not override browser chrome shortcuts; shortcut is in-textarea only.

RB-12

RB-20

Inverse success criterion — the best shortcut is the one users never think about. If we ship Cmd+Enter and three weeks later no one mentions it, we won.

Convergence

The broad field collapses into one additive shortcut, one shared seam, existing actions and existing guards.

Abandoned first passThe first convergence proposed a useComposerSubmitShortcut hook. Source inspection overturned it: SlashPickerTextarea already owns the picker state needed to arbitrate Enter, so the final direction below extends that component instead of adding a hook or file.
Source-grounded seam

Put the chord where picker state already lives.

Source-grounded seam: SlashPickerTextarea currently owns slash-picker open state and consumes Enter for picker selection, so avoid a global listener or a new shortcut manager/hook. Add one optional submit-shortcut callback to the existing shared textarea; it can suppress the chord while its picker is open, during IME composition, or on key repeat, then each caller invokes its existing submit path and guards.

Conflict posture

Conflict posture: no browser-wide shortcut interception. React only to exact Meta-or-Control plus Enter while the relevant textarea has focus, call preventDefault only for that chord, and advertise both with a compact visible hint and aria-keyshortcuts. This minimizes browser/OS/assistive-tech collision risk without claiming conflicts are impossible.

Correctness contract

Correctness contract: keyboard and button converge on the same create/reply action, preserving blank, pending, upload, run-state, staged-control, and comment-vs-reply checks. The disabled button state remains the source of user feedback; no toast, animation, telemetry, or confirmation dialog is needed.

Verification

Verification should extend existing Playwright specs: both modifiers submit, plain and Shift+Enter preserve multiline text, open slash picker and active IME do not submit, disabled/pending/upload states do not submit, and repeated keydown cannot create duplicate requests.
